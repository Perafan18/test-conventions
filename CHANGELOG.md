## [3.0.0] - 2026-05-21

### Breaking â€” rebrand to `pinto`

The project ships under a new name. Same architecture, same fixers, same canonical doc â€” just a shorter, more memorable identity. **pinto** plays on Pint (the Laravel formatter we sit on top of) + Test, and "pinto" in Spanish means "marked / spotted" â€” which is what a linter does to non-conforming code.

- Composer package: `perafan/test-conventions` â†’ `perafan/pinto`
- Binary: `vendor/bin/test-conventions` â†’ `vendor/bin/pinto`
- Fixer namespace: `Perafan/test_conventions_*` â†’ `Pinto/*` (e.g. `Pinto/max_description_length`)
- PHP namespace: `Perafan\TestConventions\` â†’ `Perafan\Pinto\`
- Client config file: `test-conventions.php` â†’ `pinto.php`
- Claude Code plugin: `test-conventions` â†’ `pinto`
- Repo: `Perafan18/test-conventions` â†’ `Perafan18/pinto`
- ENV var for the violation collector: `TEST_CONVENTIONS_VIOLATIONS_FILE` â†’ `PINTO_VIOLATIONS_FILE`

### Migration from v2.x

```bash
composer remove --dev perafan/test-conventions
composer require --dev perafan/pinto
vendor/bin/pinto init --force
```

`init --force` rewrites the client's `.php-cs-fixer.dist.php` delegate to point at the new vendor path, and renames any existing `test-conventions.php` overrides if present.

In each rule entry under `setRules`, replace `Perafan/test_conventions_*` with `Pinto/*`. `init --force` regenerates the config from scratch; if a project relied on a custom `.php-cs-fixer.dist.php` it manually authored, the rule strings need updating by hand.

### Unchanged

- All 11 fixers, behaviour, and configuration options.
- All four CLI subcommands (`check`, `fix`, `list-rules`, `init`).
- `CONVENTIONS.md` canonical doc and the canon/local split.
- v2.x clients continue to work as long as they pin `perafan/test-conventions:^2.x`. The package is no longer maintained under that name â€” pinned consumers should plan to migrate.

## [2.0.4] - 2026-05-20

### Added

- `NoSleepFixer` now accepts an `allow_in_files` configuration option (default: `['DuskTestCase.php', 'TestCase.php', 'Pest.php']`) so legitimate `usleep()` calls in test-infrastructure files (e.g. polling for a server to come up in `DuskTestCase::setUp`) don't trigger the rule. ## [2.0.3] - 2026-05-20

### Fixed

- Use `Application::addCommands(array)` instead of `Application::add(Command)` in the CLI bootstrap. The single-command `add()` method was removed in Symfony Console v8. The array-based `addCommands()` works on both v7 and v8. Without this fix, projects that resolve to Symfony Console v8 (Laravel â‰Ą 12 + recent installs) hit `Call to undefined method Application::add()` on first invocation of `vendor/bin/pinto`.

## [2.0.2] - 2026-05-20

### Fixed

- Loosened `symfony/console` and `symfony/process` constraints from `^7.0` to `^7.0 || ^8.0`. The `^7.0`-only constraint forced consumer projects (which often run on Symfony 8 via Laravel â‰Ą 12) to downgrade those components when installing this package, breaking Laravel Dusk's `ChromeProcess` (`TimeoutException` in Browser tests) and other code paths that depend on Symfony 8 behaviour.
- Tested under both Symfony 7 and Symfony 8 installations; package suite passes (57 tests, 67 assertions) on each.

## [2.0.1] - 2026-05-20

## [2.0.0] - 2026-05-20

### Breaking â€” new CLI surface

- Adds `vendor/bin/pinto` binary with subcommands `check`, `fix`, `list-rules`, `init`.
- The recommended client setup is now `composer require --dev perafan/pinto && vendor/bin/pinto init`.
- The client's `.php-cs-fixer.dist.php` (generated by `init`) is now a single line delegating to `vendor/perafan/pinto/php-cs-fixer.config.php`. The package owns the config; the client owns overrides via `pinto.php`.
- CI step in client repos changes from `vendor/bin/php-cs-fixer fix --dry-run -vvv` to `vendor/bin/pinto check`. Output is compiler-style (`file:line: rule: message`), no `-vvv` necessary.

### Internal

- Fixers that previously reported violations via `throw new RuntimeException(...)` now use the new `$this->report($file, $line, $message)` helper. Behaviour is identical when the binary is not in use (still throws), so backward compatibility is preserved for clients that keep their own `.php-cs-fixer.dist.php`. Inside the binary, a file-based `ViolationCollector` aggregates results, so a single `check` reports all violations of all rules in all files (no longer just the first error per file).
- New: `src/Cli/{Application, CheckCommand, FixCommand, ListRulesCommand, InitCommand, PhpCsFixerRunner, ViolationCollector, Violation, AutofixDiff}.php`.
- New: `php-cs-fixer.config.php` at package root â€” the single source of truth for the fixer config. Reads optional `pinto.php` from `getcwd()` to apply client overrides.
- Updated: `templates/.php-cs-fixer.dist.php` is now the 1-line delegate.
- Updated: `templates/pinto.php` has commented-out overrides for the most common knobs.

### Migration from v1.x

Existing clients using the v1.x style (full `.php-cs-fixer.dist.php` with `registerCustomFixers(...)` inline) continue to work â€” backward compatible. To opt in to the new UX:

1. `composer update perafan/pinto`.
2. `vendor/bin/pinto init --force` (overwrites the existing `.php-cs-fixer.dist.php` with the 1-line delegate; reuses any existing `pinto.php`).
3. Replace `vendor/bin/php-cs-fixer fix --dry-run` with `vendor/bin/pinto check` in CI and hooks.
